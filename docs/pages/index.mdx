# Introduction to Soxbase Commerce

A fully featured next gen commerce platform, with support for Magento 2 and GraphCMS.

## Development goals

- **boring**: Less magic is more important than absolutely minimal code
- **pleasant**: Developing should be straightforward and should follow a singluar mindset
- **strict**: Types, linters and formatters over developer knowledge
- **polymorphic**: Every backend service should be a first class citizen
- **composable**: Support customizations to backends without sacraficing maintainability
- **automation**: Every step that can be automated, should

### Library goals

- **chosen**: Libraries should be chosen carefully to solve a general issue
- **small**: Libraries should be as small as possible while offering maximum value
- **popular**: Libraries should be popular and on the rise
- **exciting**: Libraries should be exciting to work with

## Website goals

- **fast**: Performance is more important than every possible feature
- **beautiful**: Deliverying a beautiful user experience makes visitors customers
- **playful**: Animations should be smooth and easy and should guide users in their interactions
- **mobile**: Everything should work exceptionally on mobile

## Magento goals

- **compatible**: Always be compatible with the default Magento Open Source installation
- **complete**: Support all available GraphQL API's available

## Libraries and Technologies

Soxbase commerce consists of very carefully selected libraries to compose it's architecture.
Maintaining code is hard and therefor selecting third party libraries that do the heavy lifting is
vital.
